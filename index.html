Atelier elasticsearch.

0. Prérequis

  • Avoir un mongodb en mode replica set (obligatoire pour la river mongodb). Vous pouvez trouver une doc ici
  • Importer le fichier de données (ici) dans la base test-es, collection customers
Solution 
$ wget http://fastdl.mongodb.org/osx/mongodb-osx-x86_64-2.4.1.tgz
$ tar -xvf mongodb-osx-x86_64-2.4.1.tgz
$ cd mongodb-osx-x86_64-2.4.1/bin/
$ mongod --replSet rs0
$ mongo
> rs.initiate()
> exit
$ tar -xvf ~/Downloads/data.tgz
$ mongoimport -db test-es --collection customers --file test.json

1. Installer elasticsearch

Vous pouvez télécharger eleasticsearch 0.20.6 ici. Attention ! La version 0.90.0.RC1 n’est pas compatible avec tous les plugins.

Solution 
$ wget https://download.elasticsearch.org/elasticsearch/elasticsearch/elasticsearch-0.20.6.tar.gz
$ tar -xvf elasticsearch-0.20.6.tar.gz

2. Installer des plugins

Pour l'atelier, nous allons utiliser les plugins suivants :
  • head : Homepage
  • bigdesk : Homepage
  • mongodb river : Homepage
Solution 
$ cd elasticsearch-0.20.6/bin
$ ./plugin -install mobz/elasticsearch-head
$ ./plugin -install lukas-vlcek/bigdesk
$ ./plugin -install elasticsearch/elasticsearch-mapper-attachments/1.6.0
$ ./plugin -url https://github.com/downloads/richardwilly98/elasticsearch-river-mongodb/elasticsearch-river-mongodb-1.6.4.zip -install river-mongodb

4. Afficher le contenu de l’index

Pour s'assurer que les documents sont bien dans l'index, afficher le contenu de l'index foo.

Hint
Vous pouvez trouver la documentation de l'api search ici
Solution 
$curl -XGET 'http://localhost:9200/foo/' -d '{
  "query" : {
    "match_all" : {}
  }
}'
ou encore plus simple 
$curl -XGET 'http://localhost:9200/foo/'

5. Chercher des documents

Faire une recherche pour trouver le document qui contient America en faisant une requête dans l'url et une requête en JSON

Hint
Vous pouvez trouver la documentation de la query DSL ici
Solution 
$ curl -XPOST 'http://localhost:9200/foo/bar/_search?q=text:America'
$ curl -XPOST 'http://localhost:9200/foo/bar/_search' -d '{
  "query" : {
    "match" : {
      "text" : "America"
    }
  }
}'

6. Formatter la réponse

Faire la recherche précédente tout en formattant la réponse pour qu'elle soit lisible humainement.

Solution 
$ curl -XPOST 'http://localhost:9200/foo/bar/_search?pretty=true&q=text:America'
$ curl -XPOST 'http://localhost:9200/foo/bar/_search?pretty=true' -d '{
  "query" : {
    "match" : {
      "text" : "America"
    }
  }
}'

7. Initialiser la river mongodb

Initialiser la river mongodb pour créer l'index bank, avec des documents de type customers

Hint
Vous pouvez trouver de la documentation ici
Solution 
$ curl -XPOST 'http://localhost:9200/foo/bar/_search?pretty=true' -d '{
  "type": "mongodb",
    "mongodb": {
      "db": "test-es", 
      "collection": "customers"
    },
    "index": {
      "name": "bank", 
      "type": "customers"
    }
}'
L'activation de la river mongodb se manifest avec des logs dans la console. On peut aussi voir l'activité sur le plugin bigdesk. L'indexation doit prendre moins de 5 minutes.
